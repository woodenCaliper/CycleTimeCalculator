# AMR サイクルタイム シミュレーター 設計ドキュメント

## 1. 概要

### 1.1 目的
AMR（Autonomous Mobile Robot）を顧客拠点に導入する際、実際の倉庫レイアウト上でAMRの動きを視覚的にシミュレーションし、サイクルタイムとスループットを計測するツールを提供する。営業・エンジニアが顧客提案時に「この倉庫レイアウトで、何台のAMRを投入すれば、どの程度のスループットが見込めるか」を視覚的に確認・提案できることを目指す。

### 1.2 対象ユーザー
- セールスエンジニア（顧客提案時のデモ・概算見積もり）
- 導入エンジニア（レイアウト検討時のサイクルタイム試算）
- プロジェクトマネージャー（台数・ROI検討）

### 1.3 技術スタック
- フロントエンド: HTML + CSS + JavaScript（単一ファイル、サーバー不要）
- 描画エンジン: HTML5 Canvas
- シミュレーション: JavaScript によるステップベース離散イベントシミュレーション
- 外部ライブラリ: なし（純粋なVanilla JS）

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| Station | AMRが台車とドッキング/リリースする地点。Pickup と DropOff の2種類がある |
| Pickup Station | AMRが台車を受け取る（ドッキングする）地点 |
| DropOff Station | AMRが台車を届ける（リリースする）地点 |
| 待機エリア（Home） | AMRがタスクを持たない時に待機する場所 |
| 経路（Path） | Station・待機エリア間を結ぶ走行可能な線分の集合（折れ線） |
| ノード（Node） | 経路の分岐点・接続点。Station・待機エリア・経路の中間点を含む |
| セグメント（Segment） | 2つのノードを結ぶ直線区間 |
| サイクルタイム | AMRが1つのタスク（Pickup→DropOff→Home復帰）を完了するまでの総時間 |
| ドッキング | AMRが台車と結合する作業 |
| リリース | AMRが台車を切り離す作業 |
| 交差点 | 2つ以上のセグメントが交わる点。AMR同士の干渉制御が必要 |

## 3. ユーザー操作フロー

### 3.1 全体フロー

```
Step 1: 背景画像の読み込み
  └→ 倉庫レイアウト画像をアップロード → Canvasの背景に表示

Step 2: ノードの配置
  └→ Pickup Station / DropOff Station / 待機エリア をビュアー上にクリックで配置
  └→ 必要に応じて経路の中間ノード（Waypoint）も配置

Step 3: 経路の作成
  └→ ノード同士をクリックで接続し、走行経路（折れ線）を作成
  └→ 接続されたノード間がAMRの走行可能な区間となる

Step 4: パラメータの設定
  └→ AMRの走行速度
  └→ ドッキング時間（Pickup Stationでの待機時間）
  └→ リリース時間（DropOff Stationでの待機時間）
  └→ AMR台数
  └→ スケール設定（画像上の距離 → 実距離の換算比率）

Step 5: シミュレーション実行
  └→ Startボタンで開始
  └→ AMRがCanvas上で動くアニメーション
  └→ タイマー表示、搬送完了数カウント
  └→ 平均サイクルタイム・スループットをリアルタイム表示
```

### 3.2 各ステップの詳細

#### Step 1: 背景画像の読み込み
- ファイル選択ダイアログで画像を選択（PNG/JPG）
- Canvas全体に背景として描画（アスペクト比維持）
- 画像読み込み後にスケール設定UIが表示される

#### Step 2: ノードの配置
- ツールバーからノードタイプを選択 → Canvas上をクリックで配置
- ノードタイプ:
  - **Pickup Station**（緑色の四角アイコン）: 台車ドッキング地点
  - **DropOff Station**（赤色の四角アイコン）: 台車リリース地点
  - **待機エリア / Home**（青色の丸アイコン）: AMR待機場所
  - **Waypoint**（グレーの小丸）: 経路の中間点・折り返し点
- 配置後、ドラッグで位置調整が可能
- 右クリックで削除

#### Step 3: 経路の作成
- 「経路作成モード」に切り替え
- ノードを順番にクリックして折れ線を作成
- 最後のノードをダブルクリックで経路確定
- **Escapeキーで経路作成を中断**（それまでに確定済みのセグメントは残る。未確定のプレビュー線のみ破棄）
- 同一ノードから複数の経路を分岐可能（分岐＝交差点として扱う）
- 経路上にセグメントの距離（m）を表示

#### Step 4: パラメータ設定
- サイドパネルで以下を入力

#### Step 5: シミュレーション実行
- Start/Pause/Reset ボタン
- シミュレーション速度の調整（1x, 2x, 5x, 10x）
- リアルタイムで統計情報を表示

## 4. データモデル

### 4.1 ノード（Node）

```javascript
{
  id: "node_001",
  type: "pickup" | "dropoff" | "home" | "waypoint",
  x: 350,          // Canvas上のX座標 (px)
  y: 200,          // Canvas上のY座標 (px)
  label: "Pickup A" // 表示名（任意）
}
```

### 4.2 セグメント（Segment）

```javascript
{
  id: "seg_001",
  from: "node_001",  // 始点ノードID
  to: "node_002",    // 終点ノードID
  distance: 0        // 実距離 (m) — スケール係数から自動計算
}
```

セグメントは双方向通行可能とする。

### 4.3 経路グラフ（Graph）

ノードとセグメントの集合で無向グラフを構成する。AMRの経路探索はこのグラフ上でダイクストラ法により最短経路を求める。

```
Graph = {
  nodes: Map<nodeId, Node>,
  segments: Map<segmentId, Segment>,
  adjacency: Map<nodeId, Set<nodeId>>  // 隣接リスト
}
```

### 4.4 AMR 状態

```javascript
{
  id: "amr_001",
  state: "idle" | "moving_to_pickup" | "docking" | "moving_to_dropoff" | "releasing" | "returning_home" | "waiting",
  currentSegment: "seg_003",  // 現在走行中のセグメント
  positionOnSegment: 0.65,    // セグメント上の位置 (0.0〜1.0)
  x: 0, y: 0,                // Canvas上の現在座標（描画用）
  currentPath: ["node_01", "node_03", "node_05"],  // 現在のルート
  pathIndex: 1,               // currentPath上の次の目標ノードindex
  speed: 0,                   // 現在速度 (m/s)
  hasCargo: false,            // 台車を牽引中か
  taskCount: 0,               // 完了したタスク数
  totalCycleTime: 0,          // 累積サイクルタイム（平均計算用）
  currentCycleStart: 0,       // 現在のサイクル開始時刻
  waitTimer: 0                // Station/待機の残り時間
}
```

### 4.5 シミュレーション設定

```javascript
{
  scale: 10,                // 1px = X m（スケール係数）
  amrSpeed: 1.0,            // 走行速度 (m/s)
  dockingTime: 30,          // ドッキング作業時間 (秒)
  releaseTime: 20,          // リリース作業時間 (秒)
  amrCount: 1,              // AMR台数
  simSpeed: 1               // シミュレーション速度倍率
}
```

## 5. シミュレーションエンジン

### 5.1 ステップベースシミュレーション

シミュレーションは固定タイムステップ（dt = 0.1秒）で進行する。

```
毎ステップ:
  for each AMR:
    switch (AMR.state):
      case "idle":
        → 空いている Pickup Station を探す
        → 見つかれば最短経路を計算して "moving_to_pickup" へ遷移
        → 見つからなければ "idle" のまま待機

      case "moving_to_pickup":
        → 経路に沿って移動（衝突回避あり）
        → Pickup Station到着 → "docking" へ遷移、タイマー開始

      case "docking":
        → タイマー消化（dockingTime秒待つ）
        → 完了 → hasCargo = true
        → 最寄りの DropOff Station への最短経路を計算
        → "moving_to_dropoff" へ遷移

      case "moving_to_dropoff":
        → 経路に沿って移動（衝突回避あり）
        → DropOff Station到着 → "releasing" へ遷移、タイマー開始

      case "releasing":
        → タイマー消化（releaseTime秒待つ）
        → 完了 → hasCargo = false, taskCount++
        → サイクルタイム記録
        → Home への最短経路を計算
        → "returning_home" へ遷移

      case "returning_home":
        → 経路に沿って移動（衝突回避あり）
        → Home到着 → "idle" へ遷移

      case "waiting":
        → 衝突回避による一時停止中
        → 前方が空いたら元の状態に復帰
```

### 5.2 移動処理

各ステップでのAMR移動:

```
移動距離 = amrSpeed * dt * simSpeed (m)
ピクセル移動距離 = 移動距離 / scale (px)
```

AMRは currentPath に従い、ノード間をセグメントに沿って直線移動する。

### 5.3 衝突回避（交差点制御）

複数AMRが同時に走行する場合の干渉を防ぐために、**セグメントベースの排他制御**を採用する。

#### ルール
1. **セグメント占有**: 各セグメントには同時に1台のAMRしか進入できない
2. **予約制**: AMRは次に進入するセグメントを事前に予約する
3. **優先度**: セグメントが競合した場合、以下の優先順位で解決する:
   - 荷物を持っているAMR（搬送中）が優先
   - 同じ優先度なら、先に予約したAMR が優先
   - それでも同じなら、AMR IDが小さい方が優先
4. **待機**: 優先度が低いAMRは、セグメントが空くまで現在位置で停止する

#### 実装

```javascript
// セグメント予約テーブル
segmentReservation = Map<segmentId, amrId | null>

// AMRが次のセグメントに進入する前に:
function tryReserveSegment(amrId, segmentId) {
  if (segmentReservation[segmentId] === null) {
    segmentReservation[segmentId] = amrId;
    return true;  // 予約成功、進入可能
  }
  if (segmentReservation[segmentId] === amrId) {
    return true;  // 自分が既に予約済み
  }
  return false;   // 他のAMRが占有中 → 待機
}

// AMRがセグメントを通過完了したら:
function releaseSegment(amrId, segmentId) {
  if (segmentReservation[segmentId] === amrId) {
    segmentReservation[segmentId] = null;
  }
}
```

### 5.4 経路探索（ダイクストラ法）

AMRが目的地（Station/Home）へ向かう際、グラフ上の最短経路をダイクストラ法で求める。

```
入力: 始点ノード, 終点ノード（の集合）, グラフ
出力: 最短経路（ノードIDの配列）
重み: セグメントの距離（m）
```

DropOff Stationが複数ある場合、全DropOffの中で最も近いものを選択する。Homeも同様。

### 5.5 タスク割り当て

シンプルなFIFO方式:
1. idle状態のAMRは、Pickup Stationのリストを巡回
2. 他のAMRがまだ割り当てられていないPickup Stationを選ぶ（最寄り優先）
3. 全Pickup Stationが使用中なら、Homeで待機

### 6.5 シミュレーション制御ボタン

3つのボタンで構成する:

```
  [▶ / ⏸]     [⏩ x3]      [⏹ Reset]
   再生/一時停止   早送り        リセット
```

#### ① 再生/一時停止ボタン（トグル動作）

| 現在の状態 | ボタン表示 | クリック時の動作 |
|-----------|----------|----------------|
| 停止中（初期状態） | ▶ 再生 | シミュレーション開始（1倍速） |
| 再生中 | ⏸ 一時停止 | シミュレーション一時停止 |

| 早送り中 | ⏸ 一時停止 | シミュレーション一時停止 |

#### ② 早送りボタン

- ボタン横に数値入力欄を設置し、倍速の値をユーザーが自由に入力できる（デフォルト: 3）
- 入力範囲: 2〜100（整数）

| 現在の状態 | ボタン表示 | クリック時の動作 |
|-----------|----------|----------------|
| 1倍速で再生中 | ⏩ x3 | 入力された倍速に切り替え（例: 3倍速） |

| 一時停止中 | ⏩ x3 | 入力された倍速で再開 |

※ 早送り中にもう一度押すと1倍速に戻る（トグル動作）

### 7.4 シミュレーションループ

```javascript
function simulationLoop(timestamp) {
  if (!running) return;

  const dtReal = (timestamp - lastTimestamp) / 1000;  // 実時間差分(秒)
  lastTimestamp = timestamp;

  const dtSim = dtReal * simSpeed;  // シミュレーション時間差分

  // ステップ実行（dtSimを固定ステップに分割して精度確保）
  const fixedDt = 0.05;  // 50ms固定ステップ
  let remaining = dtSim;
  while (remaining > 0) {
    const step = Math.min(remaining, fixedDt);
    simulationEngine.update(step);
    remaining -= step;
  }

  // 描画
  canvasRenderer.render();

  // 統計更新
  uiController.updateStats();

  requestAnimationFrame(simulationLoop);
}
```

## 8. 衝突回避の詳細設計

### 8.1 セグメント占有モデル

各セグメントは「1車線の一方通行道路」として扱い、同時に1台のAMRのみが占有できる。

```
セグメントの状態:
  - FREE: 空き（進入可能）
  - OCCUPIED(amr_id): AMRが占有中
  - RESERVED(amr_id): AMRが次に進入予定
```

### 8.2 デッドロック回避

2台のAMRが互いに相手の占有セグメントに進入しようとすると、デッドロックが発生しうる。これを回避するため:

1. **タイムアウト検出**: 待機が一定時間（10秒）を超えたらデッドロックの可能性ありと判定
2. **優先度による退避**: 優先度の低いAMRが経路を再計算し、別ルートを探す
3. **別ルートがない場合**: 優先度の低いAMRが現在のセグメントを解放して元のノードに戻る

### 8.3 交差点の定義

グラフ上で次数（接続セグメント数）が3以上のノードを「交差点」と定義する。交差点に接続するセグメント群は連動して排他制御される：

```
交差点ノードに進入する場合:
  → 交差点ノードに接続する全セグメントのうち、
    自分が使用する入口セグメント＋出口セグメントの両方を予約する必要がある
  → どちらか一方でも予約できなければ、手前のノードで待機
```

## 9. スケール設定

### 9.1 スケール係数の設定方法

ユーザーが画像上の既知の距離をもとにスケールを校正する:

1. 「スケール設定」ボタンを押す
2. Canvas上で2点をクリック
3. 「この2点間の実際の距離は何メートルですか？」と表示
4. 数値入力 → スケール係数を自動計算

```
scale (m/px) = 入力距離(m) / 2点間のピクセル距離(px)
```

### 9.2 距離計算

セグメントの実距離:
```
distance(m) = sqrt((x2-x1)² + (y2-y1)²) * scale
```

## 10. 出力・統計

### 10.1 リアルタイム統計

| 指標 | 計算式 |
|------|--------|
| 経過時間 | シミュレーション開始からの累積時間 |
| 搬送完了数 | 全AMRの taskCount 合計 |
| 平均サイクルタイム | Σ(各サイクルの所要時間) / 搬送完了数 |
| スループット | 搬送完了数 / 経過時間(時) |
| AMR別待機率 | AMRが "waiting" 状態だった時間 / 経過時間 |

### 10.2 サイクルタイムの内訳記録

各サイクルについて以下を記録:
```javascript
{
  amrId: "amr_001",
  cycleNumber: 3,
  pickupStation: "Pickup A",
  dropoffStation: "DropOff B",
  travelToPickup: 45.2,    // 秒
  dockingTime: 30.0,        // 秒
  travelToDropoff: 62.8,    // 秒
  releaseTime: 20.0,        // 秒
  returnToHome: 38.5,       // 秒
  waitingTime: 8.3,         // 秒（衝突回避による待機の累計）
  totalCycleTime: 204.8     // 秒
}
```

## 11. 前提条件・制約

- AMRは全台同一仕様（速度・ドッキング時間は共通）
- セグメントは双方向通行可能だが、同時に1台のみ占有
- AMRの物理的な寸法は考慮しない（点として扱う）
- 加速・減速は省略し、一定速度で走行するモデルとする（Phase 1）
- Pickup Stationには常に台車が供給されている（台車の在庫切れは考慮しない）
- DropOff Stationは常に台車を受け入れ可能（満杯は考慮しない）
- 経路はユーザーが手動で作成（自動生成なし）
