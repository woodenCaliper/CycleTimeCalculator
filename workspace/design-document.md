# AMR サイクルタイム シミュレーター 設計ドキュメント

## 1. 概要

### 1.4 重要変更（最新版優先）
- 本版より**経路作成は自由接続**とする。
- 旧版に記載されていた「単一ループ必須」「ループ内次数制限」「ループ完成前提」の記述は、
  本版の「自由接続グラフ」仕様よりも優先度が低い（読み替え対象）。
- AMRの経路探索は、ループトレースではなく**接続グラフ上の到達可能経路探索**を基準とする。

### 1.1 目的
AMR（Autonomous Mobile Robot）を顧客拠点に導入する際、実際の倉庫レイアウト上でAMRの動きを視覚的にシミュレーションし、サイクルタイムとスループットを計測するツールを提供する。営業・エンジニアが顧客提案時に「この倉庫レイアウトで、何台のAMRを投入すれば、どの程度のスループットが見込めるか」を視覚的に確認・提案できることを目指す。

### 1.2 対象ユーザー
- セールスエンジニア（顧客提案時のデモ・概算見積もり）
- 導入エンジニア（レイアウト検討時のサイクルタイム試算）
- プロジェクトマネージャー（台数・ROI検討）

### 1.3 技術スタック
- フロントエンド: HTML + CSS + JavaScript（単一ファイル、サーバー不要）
- 描画エンジン: HTML5 Canvas
- シミュレーション: JavaScript によるステップベース離散イベントシミュレーション
- 外部ライブラリ: なし（純粋なVanilla JS）

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| Station | AMRが台車とドッキング/リリースする地点。Pickup と DropOff の2種類がある |
| Pickup Station | AMRが台車を受け取る（ドッキングする）地点 |
| DropOff Station | AMRが台車を届ける（リリースする）地点 |
| 待機エリア（Home） | AMRがタスクを持たない時に待機する場所 |
| 経路（Path） | AMRが走行するルート全体。Station / Waypoint / Home を任意接続したグラフとして構成される |
| 経路作成 | ノード同士を自由に直線セグメントで接続して作成する。閉路（ループ）である必要はない |
| Home接続 | Waiting Area（Home）ノードと他ノードを結ぶ接続。必要に応じて経路作成ツールまたはHome枝接続ツールで作成可能 |
| ノード（Node） | 経路上の接続点。Station・Waypoint・Waiting Areaを含む |
| セグメント（Segment） | 2つのノードを結ぶ直線区間 |
| 経路属性（Direction Attribute） | 各セグメントに付与される通行方向属性。`A↔B（双方向）` / `A→B（片方向）` / `B→A（片方向）` の3種 |
| SyncArea | ユーザーが直線で囲って作成する多角形エリア。エリア内には同時に1台のAMRのみ侵入可能 |
| サイクルタイム | AMRが1往復分のタスク（Pickup到着→ドッキング→DropOff到着→リリース→次のPickup到着）を完了するまでの総時間。HomeへのReturn移動はサイクルに含まない |
| ドッキング | AMRが台車と結合する作業 |
| リリース | AMRが台車を切り離す作業 |

## 3. ユーザー操作フロー

### 3.1 全体フロー

```
Step 1: 背景画像の読み込み
  └→ 倉庫レイアウト画像をアップロード → Canvasの背景に表示

Step 2: ノードの配置
  └→ Pickup Station / DropOff Station / 待機エリア をビュアー上にクリックで配置
  └→ 必要に応じて経路の中間ノード（Waypoint）も配置

Step 3: 経路の作成
  └→ ノード同士をクリックで自由に接続し、走行経路（折れ線）を作成
  └→ 経路は閉路（ループ）である必要はない
  └→ 必要に応じて SyncArea（排他進入エリア）を多角形で作成

Step 4: パラメータの設定
  └→ AMRの走行速度
  └→ ドッキング時間（Pickup Stationでの待機時間）
  └→ リリース時間（DropOff Stationでの待機時間）
  └→ AMR台数
  └→ Pickup供給周期（例: 20秒ごとに台車1台を補充）
  └→ スケール設定（画像上の距離 → 実距離の換算比率）

Step 5: シミュレーション実行
  └→ Startボタンで開始
  └→ AMRがCanvas上で動くアニメーション
  └→ タイマー表示、搬送完了数カウント
  └→ 平均サイクルタイム・スループットをリアルタイム表示
```

### 3.2 各ステップの詳細

#### Step 1: 背景画像の読み込み
- ファイル選択ダイアログで画像を選択（PNG/JPG）
- Canvas全体に背景として描画（アスペクト比維持）
- 画像読み込み後にスケール設定UIが表示される

#### Step 2: ノードの配置
- ツールバーからノードタイプを選択 → Canvas上をクリックで配置
- ノードタイプ:
  - **Pickup Station**（緑色の四角アイコン）: 台車ドッキング地点
  - **DropOff Station**（赤色の四角アイコン）: 台車リリース地点
  - **待機エリア / Home**（青色の丸アイコン）: AMR待機場所（**1ノードあたり同時に1台のみ停止可**）
  - **Waypoint**（グレーの小丸）: 経路の中間点・折り返し点
- 配置後、ドラッグで位置調整が可能
- 右クリックで削除

#### Step 3: 経路の作成

経路の作成は **①経路セグメントの作成** と **②SyncAreaの作成** を必要に応じて行う。

##### ① 経路セグメントの作成（自由接続）
- 「経路作成モード」に切り替え
- 経路属性セレクタで `A↔B / A→B / B→A` を選択してからセグメントを作成
- ノードを順番にクリックして折れ線を作成
- 始点を再クリックすると、その時点までの作成を確定する（閉路は不要）
- **Escapeキーで経路作成を中断**（作成中のセグメントをすべて破棄）
- Homeノードを含め、任意のノード間を接続できる
- 一方通行セグメントはクリック順を `A→B` のA/Bとして解釈する（逆方向を許可したい場合は `B→A` を選択）

##### ② Home枝の接続（オプション）
- 必要に応じて「Home枝接続モード」で操作する
- 任意の非Homeノードをクリック → 接続元として選択
- Waiting Area（Home）ノードをクリック → Home枝セグメントを作成
- Home枝にも同じ経路属性（双方向 / 片方向）を設定できる
- 1つのノードから複数のHome接続を延ばすことも可能
- Waiting Areaノード同士を直接接続することは不可
- **待機エリア占有制約**: 同一Waiting Areaへ複数AMRを同時に帰還させない（帰還先は空き/予約なしのHomeから選択）

- 各セグメント中点に距離（m）を表示
- 双方向セグメントは従来通りの線表示、片方向セグメントは進行方向の矢印を重ねて表示
- 経路が1本もない状態ではシミュレーション開始不可（UIでエラー表示）

##### ③ SyncAreaの作成（排他進入エリア）
- 「SyncArea作成モード」に切り替え
- Canvas上をクリックして頂点を追加し、始点を再クリックして多角形を確定
- SyncAreaは直線辺のみで構成される閉領域（3頂点以上）
- **制約**: 同一SyncArea内には同時に1台のAMRのみ侵入可能
- 先行AMRがSyncArea内にいる場合、後続AMRは境界外で待機する
- 待機AMRはSyncArea進入待ちの間、必要に応じて進入前セグメント予約を解放し、先行AMRの退出を妨げない

#### Step 4: パラメータ設定
- サイドパネルで以下を入力
  - 速度
  - ドッキング時間
  - リリース時間
  - AMR台数
  - Pickup供給周期（秒）
  - スケール

#### Step 5: シミュレーション実行
- Start/Pause/Reset ボタン
- シミュレーション速度の調整（1x, 2x, 5x, 10x）
- リアルタイムで統計情報を表示

## 4. データモデル

### 4.1 ノード（Node）

```javascript
{
  id: "node_001",
  type: "pickup" | "dropoff" | "home" | "waypoint",
  x: 350,          // Canvas上のX座標 (px)
  y: 200,          // Canvas上のY座標 (px)
  label: "Pickup A" // 表示名（任意）
}
```

### 4.2 セグメント（Segment）

```javascript
{
  id: "seg_001",
  from: "node_001",  // 始点ノードID
  to: "node_002",    // 終点ノードID
  direction: "bidirectional" | "forward" | "reverse", // forward: from→to, reverse: to→from
  distance: 0        // 実距離 (m) — スケール係数から自動計算
}
```

`direction` により通行可否を決定する。

- `bidirectional`: `from→to` と `to→from` の両方向通行可
- `forward`: `from→to` のみ通行可
- `reverse`: `to→from` のみ通行可

### 4.3 経路グラフ（Graph）

ノードとセグメントの集合で**任意接続可能な経路グラフ**を構成する。

#### グラフのトポロジー

```
                 [Waypoint]
                /          \
[PickupA] ----              ---- [DropOffB]
                \          /          |
                [PickupB]          [Home 1]  ← Home枝（リーフ）
                    |
                 [Home 2]  ← Home枝（リーフ）
```

- ノードは任意接続できる（閉路・非閉路どちらも可）
- Homeノードは必要に応じて他ノードに接続して待機地点として利用する

#### 接続制約

| 対象ノード | 接続できるセグメント数 |
|-----------|---------------------|
| Pickup / DropOff / Waypoint | 任意本数 |
| Home（Waiting Area） | 任意本数 |

- 経路は閉路である必要はない
- シミュレーション可能条件は「必要なノード間に到達可能な経路が存在すること」

#### データ構造

```javascript
Graph = {
  nodes: Map<nodeId, Node>,
  segments: Map<segmentId, Segment>,
  adjacency: Map<nodeId, Set<nodeId>>,  // 全隣接リスト
  loopSegments: Set<segmentId>,         // ループ部を構成するセグメントのID集合
  homeSegments: Set<segmentId>,         // Home枝を構成するセグメントのID集合
  loopOrder: nodeId[]                   // ループ部ノードの順序配列（経路トレース用）
}
```

### 4.7 SyncArea

```javascript
SyncArea = {
  id: "sa_001",
  points: [{x: 100, y: 100}, {x: 220, y: 120}, {x: 180, y: 260}],
  occupantAmrId: null | "amr_001"
}
```

- `occupantAmrId` が `null` の場合のみ、新たなAMRの侵入を許可する
- 侵入後は当該AMRがエリア外へ出るまで占有を維持する
- 複数SyncAreaにまたがる場合は、侵入対象のすべてで空きであることが必要

#### バリデーション

セグメント追加時に以下を検証し、違反する場合は追加をブロックする。

```
【経路セグメント追加時】
1. 同一ノード同士の接続ではないこと
2. 同一ペア間に既存セグメントがないこと
3. 片方向属性は作成順（from/to）に対して適用すること

【Home枝接続モード使用時】
1. 接続先ノードが home タイプであること
2. 接続元は home 以外のノードであること
```

### 4.4 AMR 状態

#### AMRの状態種別

| 状態 | 説明 | 遷移先 |
|------|------|--------|
| `at_home` | WaitingArea待機中（初回スタート前またはタスクなし時のみ） | `moving_to_pickup` |
| `moving_to_pickup` | Pickup Stationへ移動中（Home出発時 or DropOff完了後） | `docking` |
| `docking` | Pickup Stationでドッキング作業中 | `moving_to_dropoff` |
| `moving_to_dropoff` | DropOff Stationへ移動中（ループ経路上を走行） | `releasing` |
| `releasing` | DropOff Stationでリリース作業中 | `moving_to_pickup` |
| `waiting` | 衝突回避による一時停止中 | （前の状態に復帰） |
| `returning_home` | タスクなし時に空きHomeへ帰還中（例外的） | `at_home` |

> **通常サイクル**: `moving_to_pickup` → `docking` → `moving_to_dropoff` → `releasing` → `moving_to_pickup` → ...
>
> **Home帰還条件**: すべてのPickupが「在庫なし or 他AMR割当済み」の場合、または到着Pickupの在庫が0だった場合
> **Pickup割当ルール**: 他AMRの割当として扱うのは `moving_to_pickup` 中のみ。`docking` / `moving_to_dropoff` / `releasing` 中は次Pickupの割当をブロックしない。

```javascript
{
  id: "amr_001",
  state: "at_home" | "moving_to_pickup" | "docking" | "moving_to_dropoff" | "releasing" | "returning_home" | "waiting",
  currentSegment: "seg_003",  // 現在走行中のセグメント
  positionOnSegment: 0.65,    // セグメント上の位置 (0.0〜1.0)
  x: 0, y: 0,                // Canvas上の現在座標（描画用）
  currentPath: ["node_01", "node_03", "node_05"],  // 現在のルート
  pathIndex: 1,               // currentPath上の次の目標ノードindex
  hasCargo: false,            // 台車を牽引中か
  taskCount: 0,               // 完了したタスク数
  totalCycleTime: 0,          // 累積サイクルタイム（平均計算用）
  currentCycleStart: 0,       // 現在のサイクル開始時刻（Pickup到着時刻）
  waitTimer: 0,               // Station作業の残り時間
  assignedPickup: null,       // 割り当て済みPickup StationのノードID（moving_to_pickup中のみ有効）
  assignedDropoff: null,      // 割り当て済みDropOff StationのノードID
  targetHomeId: null          // 帰還先として予約済みのWaiting AreaノードID（1台制約用）
}
```

### 4.5 シミュレーション設定

```javascript
{
  scale: 10,                // 1px = X m（スケール係数）
  amrSpeed: 1.0,            // 走行速度 (m/s)
  dockingTime: 30,          // ドッキング作業時間 (秒)
  releaseTime: 20,          // リリース作業時間 (秒)
  amrCount: 1,              // AMR台数
  pickupSupplyInterval: 20, // Pickupの台車供給周期 (秒)
  simSpeed: 1               // シミュレーション速度倍率
}
```

### 4.6 Pickup供給モデル

Pickup Stationは「常時台車あり」ではなく、一定周期で台車が供給される。

```javascript
PickupSupply = {
  pickupNodeId: "node_001",
  availableCarts: 0 | 1,     // Phase1では1台バッファ
  nextSupplyTime: 20.0       // 次回供給予定時刻（シミュレーション秒）
}
```

- シミュレーション開始時点では各Pickupに1台が存在する。
- `pickupSupplyInterval` ごとに、在庫が0なら1台補充する（上限1）。
- AMRがPickup到着時に在庫があれば1台消費してドッキングへ遷移。
- 在庫が0ならドッキングせず、タスクなしとしてWaiting Areaへ帰還する。

## 5. シミュレーションエンジン

### 5.1 ステップベースシミュレーション

シミュレーションは固定タイムステップ（dt = 0.1秒）で進行する。

#### 通常サイクルの状態遷移

```
WaitingArea（初回のみ）
    ↓ moving_to_pickup
Pickup Station
    ↓ docking（dockingTime秒）
    ↓ moving_to_dropoff  ← ループ経路上を走行
DropOff Station
    ↓ releasing（releaseTime秒）
    ↓ moving_to_pickup   ← ループ経路上を逆走 or 同方向で折り返し
Pickup Station → （以降ループ）
```

```
毎ステップ:
  for each AMR:
    switch (AMR.state):

      case "at_home":
        → 在庫あり & 未割当の Pickup Station を探す（ループトレースで最近傍）
        → 見つかれば Home枝→ループ部の経路を計算し "moving_to_pickup" へ遷移
        → 見つからなければ "at_home" のまま待機

      case "moving_to_pickup":
        → 経路に沿って移動（衝突回避あり）
        → 進入先がSyncArea内で、他AMRが占有中なら境界外で待機
        → Pickup Station到着
          → 在庫あり: 1台消費し currentCycleStart = 現在時刻、"docking" へ遷移
          → 在庫なし: タスクなしとして "returning_home" へ遷移

      case "docking":
        → タイマー消化（dockingTime秒待つ）
        → 完了 → hasCargo = true
        → 経路グラフ上で最寄りの DropOff Station への経路を計算
        → "moving_to_dropoff" へ遷移

      case "moving_to_dropoff":
        → 経路に沿って移動（衝突回避あり）
        → 進入先がSyncArea内で、他AMRが占有中なら境界外で待機
        → DropOff Station到着 → "releasing" へ遷移、waitTimer リセット

      case "releasing":
        → タイマー消化（releaseTime秒待つ）
        → 完了:
          → hasCargo = false
          → taskCount++
          → サイクルタイム記録（currentCycleStart → 次のPickup到着まで = 次サイクルで記録）
          → 在庫あり & 未割当の Pickup Station を探す
          → 見つかれば 経路グラフ上で経路を計算し "moving_to_pickup" へ遷移  ← Homeへは戻らない
          → 見つからなければ Home枝へ戻り "returning_home" へ遷移（例外）

      case "returning_home":
        → 空き/未予約のWaiting Areaを探索して帰還経路を確定
        → 空きが無い場合は帰還待機（空きが出た時点で再探索）
        → Home枝経路に沿って移動（衝突回避あり）
        → 進入先がSyncArea内で、他AMRが占有中なら境界外で待機
        → Home到着 → "at_home" へ遷移（当該Homeを占有）

      case "waiting":
        → 衝突回避による一時停止中
        → 前方セグメントが空いたら元の状態に復帰
```

### 5.2 移動処理

各ステップでのAMR移動:

```
移動距離 = amrSpeed * dt * simSpeed (m)
ピクセル移動距離 = 移動距離 / scale (px)
```

AMRは currentPath に従い、ノード間をセグメントに沿って直線移動する。

### 5.3 衝突回避（ループ経路上の干渉制御）

1ループ経路のため交差点は存在しない。干渉は**同一セグメント上での正面衝突（対向）または追突**として発生する。これを防ぐために**セグメントベースの排他制御**を採用する。

#### ルール
1. **セグメント占有**: 各セグメントには同時に1台のAMRしか進入できない（方向問わず）
2. **予約制**: AMRは次に進入するセグメントを事前に予約する
3. **優先度**: セグメントが競合した場合、以下の優先順位で解決する:
   - 荷物を持っているAMR（搬送中）が優先
   - 同じ優先度なら、先に予約したAMR が優先
   - それでも同じなら、AMR IDが小さい方が優先
4. **待機**: 優先度が低いAMRは、セグメントが空くまで現在ノードで停止する
5. **交差点なし**: ループ経路ではノードの次数が最大2のため、交差点は発生しない

#### 実装

```javascript
// セグメント予約テーブル
segmentReservation = Map<segmentId, amrId | null>

// AMRが次のセグメントに進入する前に:
function tryReserveSegment(amrId, segmentId) {
  if (segmentReservation[segmentId] === null) {
    segmentReservation[segmentId] = amrId;
    return true;  // 予約成功、進入可能
  }
  if (segmentReservation[segmentId] === amrId) {
    return true;  // 自分が既に予約済み
  }
  return false;   // 他のAMRが占有中 → 待機
}

// AMRがセグメントを通過完了したら:
function releaseSegment(amrId, segmentId) {
  if (segmentReservation[segmentId] === amrId) {
    segmentReservation[segmentId] = null;
  }
}
```

### 5.4 経路探索（ループトレース）

単一ループ制約により、グラフは必ず1本の輪または A↔B の最小ループになる。ダイクストラ法の代わりに**ループ上の2方向探索**を行い、経由距離が短い方向を選ぶ。

```
入力: 始点ノード, 目的ノード（の集合）, loopOrder（ループ順ノード配列）
出力: 経路（ノードIDの配列）

アルゴリズム:
  1. loopOrder 上で始点ノードの位置 i を特定
  2. 「時計回り」と「反時計回り」の2方向で目的ノードを探索
  3. 最初に目的ノードに到達する方向・距離で最短経路を決定
  4. 目的ノードが複数ある場合は全目的ノードを対象に探索し、最も近いものを選択
```

#### 最小ループ（A↔B）の場合
- ノードが2つのみなので、常に1セグメントを通るだけ
- 往路・復路ともに同じセグメントを使用（方向のみ異なる）

### 5.5 タスク割り当て

AMRはDropOff完了後に**即座に次のPickupを探し、Homeへは戻らない**のが基本。

```
タスク探索のタイミング:
  1. シミュレーション開始時（at_home → moving_to_pickup）
  2. releasing 完了時（moving_to_dropoff → moving_to_pickup、またはreturning_home）

タスク選択ロジック:
  → 他のAMRがまだ割り当てていない Pickup Station を探す
  → ループ経路上の経由距離が短いものを優先（ループトレースで計算）
  → 見つかった場合: ループ経路上で経路を計算し "moving_to_pickup" へ
  → 見つからない場合: Home枝へ退避し "returning_home" へ（全Pickupが他AMR占有中のみ）
```

### 7.4 シミュレーションループ

```javascript
function simulationLoop(timestamp) {
  if (!running) return;

  const dtReal = (timestamp - lastTimestamp) / 1000;  // 実時間差分(秒)
  lastTimestamp = timestamp;

  const dtSim = dtReal * simSpeed;  // シミュレーション時間差分

  // ステップ実行（dtSimを固定ステップに分割して精度確保）
  const fixedDt = 0.05;  // 50ms固定ステップ
  let remaining = dtSim;
  while (remaining > 0) {
    const step = Math.min(remaining, fixedDt);
    simulationEngine.update(step);
    remaining -= step;
  }

  // 描画
  canvasRenderer.render();

  // 統計更新
  uiController.updateStats();

  requestAnimationFrame(simulationLoop);
}
```

## 8. 衝突回避の詳細設計

### 8.1 セグメント占有モデル

各セグメントは「方向属性（双方向/片方向）に従って通行し、同時に1台のみ占有可」として扱う。ループ経路のため交差点は存在しない。

```
セグメントの状態:
  - FREE: 空き（進入可能）
  - OCCUPIED(amr_id): AMRが占有中（方向問わず）
```

### 8.2 デッドロック回避（SyncArea）

自由接続グラフ + SyncAreaでは、次の条件でデッドロックが発生しうる。

- 先行AMRがSyncArea内から退出しようとしている
- 後続AMRがSyncArea進入セグメントを先に予約したまま境界外で待機している
- 先行AMRがその予約済みセグメントを取れず、双方が待機し続ける

これを回避するため、以下を実装する。

1. **進入前チェック**: 進入先が他AMR占有中のSyncAreaであれば、進入セグメントを予約しない
2. **待機時の予約解放**: SyncArea境界外で進入待機が継続したAMRは、保持中の進入前セグメント予約を解放する
3. **再試行**: 空きになった時点で再度セグメント予約→進入を行う

### 8.3 交差点について

自由接続グラフでは交差点を構成できるため、衝突回避はセグメント占有とSyncArea排他制御の組み合わせで扱う。

## 9. スケール設定

### 9.1 スケール係数の設定方法

ユーザーが画像上の既知の距離をもとにスケールを校正する:

1. 「スケール設定」ボタンを押す
2. Canvas上で2点をクリック
3. 「この2点間の実際の距離は何メートルですか？」と表示
4. 数値入力 → スケール係数を自動計算

```
scale (m/px) = 入力距離(m) / 2点間のピクセル距離(px)
```

### 9.2 距離計算

セグメントの実距離:
```
distance(m) = sqrt((x2-x1)² + (y2-y1)²) * scale
```

## 10. 出力・統計

### 10.1 リアルタイム統計

| 指標 | 計算式 |
|------|--------|
| 経過時間 | シミュレーション開始からの累積時間 |
| 搬送完了数 | 全AMRの taskCount 合計 |
| 平均サイクルタイム | Σ(各サイクルの所要時間) / 搬送完了数 |
| スループット | 搬送完了数 / 経過時間(時) |
| AMR別待機率 | AMRが "waiting" 状態だった時間 / 経過時間 |

### 10.2 サイクルタイムの定義と内訳

**サイクルの定義**: Pickup Station到着（ドッキング開始）から次のPickup Station到着（次ドッキング開始）までの時間。**HomeへのReturn移動はサイクルに含まない。**

```
1サイクル = ドッキング時間
           + Pickup→DropOffの移動時間（走行＋衝突待機）
           + リリース時間
           + DropOff→次Pickupの移動時間（走行＋衝突待機）
```

各サイクルについて以下を記録:
```javascript
{
  amrId: "amr_001",
  cycleNumber: 3,
  pickupStation: "Pickup A",
  dropoffStation: "DropOff B",
  dockingTime: 30.0,           // 秒（Pickup作業時間）
  travelToDropoff: 62.8,       // 秒（Pickup→DropOffの移動時間）
```

> ※ 初回（Home出発）サイクルは、Home→Pickup の移動時間も別途記録するが、平均サイクルタイムの計算には含めない（初回移動は定常状態でない）

## 11. 前提条件・制約

- AMRは全台同一仕様（速度・ドッキング時間は共通）
- セグメントは方向属性（双方向 / 片方向）に従って通行し、同時に1台のみ占有
- AMRの物理的な寸法は考慮しない（点として扱う）
- 加速・減速は省略し、一定速度で走行するモデルとする（Phase 1）
- Pickup Stationへの台車供給は周期的（`pickupSupplyInterval`秒ごと、上限1台）
- DropOff Stationは常に台車を受け入れ可能（満杯は考慮しない）
- 経路はユーザーが手動で作成（自動生成なし）
- 経路は任意接続可能（閉路・非閉路どちらも可）
- Waiting Area（Home）を含む任意ノード間の接続を許可する
- SyncAreaはユーザーが任意に作成できるが、同一エリア内のAMRは同時に1台まで
- 経路セグメントが1本もない状態ではシミュレーションを開始できない
- Homeノードが1つも存在しない場合もシミュレーション開始不可
